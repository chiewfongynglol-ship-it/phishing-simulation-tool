A security research project designed to demonstrate how credential harvesting works and to train users on identifying social engineering tactics.

##  Project Overview
This tool simulates a common "Account Lockout" phishing scenario. It is designed to help organizations test their "Human Firewall" by providing a safe, controlled environment to track user interaction with suspicious links.

##  Features
- **Realistic Credential Harvester:** A custom-built "ConnectPortal" login page that mimics corporate SSO environments.
- **Security Alert UI:** Uses psychological triggers (urgency and fear) to demonstrate how attackers manipulate user behavior.
- **Client-Side Logic:** Uses JavaScript to intercept form data safely without transmitting it to an external server.
- **Education Page:** Redirects "phished" users to an educational landing page (`education.html`) to teach them what they missed.

##  Cybersecurity Learning Objectives
- **Social Engineering:** Understanding the mechanics of a phishing attack from the attacker's perspective (Red Team).
- **Defense in Depth:** Demonstrating why technical controls (MFA, email filters) must be paired with user education (Blue Team).
- **Web Security:** Identifying how URI structure and SSL/TLS certificates (via GitHub Pages) affect user trust.
